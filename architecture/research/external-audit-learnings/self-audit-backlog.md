## What's strongly mitigated (selected, with the proof)

- **Oracle 100x-poison (Blend Feb-2026) + decimals (Morpho PAXG)** — for **risk-increasing and liquidation** flows a 100x outlier is rejected on two independent gates (deviation band capped at 50% + absolute sanity band $1B WAD), decimals always fetched live (never operator-supplied), production markets forced to cross-provider primary+anchor, spot-only single source rejected. **Important qualification (critic):** these flows **require the cross-provider anchor and fail *closed*** if it is unavailable — they do **not** fall through to a single poisoned source. The correlated-failure case (anchor *also* degrades) therefore reverts (the un-liquidatable residual, backlog #3), it does not over-borrow. Permissive *risk-decreasing* reads may use a degraded single source but cannot increase risk. So "100x rejected" = "value-extracting flows fail closed," not "always priced safely." Tests: `oracle/tolerance/bands.rs:214-302`, `dual_source.rs:68-120`; `policy.rs:37-43,67-73`.
- **Toxic liquidation spiral (Aave CRV)** — derived bonus enforces `threshold·(1+bonus) ≤ 1` at every threshold, placing the frontier at the threshold; liquidation is never HF-worsening. Certora `derived_bonus_respects_threshold`; harness `test_liquidation_strictly_improves_hf`.
- **Stale-index + flash-loan donation (Morpho $2.85M)** — structurally inverted: gates run post-pool against freshly-recorded indexes; utilization uses internal `cash`, not `token.balance` (the v3.1 fix). `cache.rs:84-114`.
- **Bad-debt socialization gaming (Morpho OZ-H01/3.5.6)** — triggers on collateral ≤$5 (not ==0, defeats leave-1-wei); decrement uses *capped* subtraction (`interest.rs:80-85`) so it can't underflow-revert.
- **Auth-tree phishing / $2.6M Morpho-class** — single governance-set aggregator, invoker-auth scoped to exactly one transfer with empty sub-tree, balance-delta validation; adversarial fuzz `adversarial_aggregator.rs` proves a zero-output aggregator is rejected with flash guard cleared + zero residual allowance.
- **Isolation-counter ratchet (Aave OZ-H01)** — liquidation reuses the same `adjust_isolated_debt_for_repay` as ordinary repay; harness `test_isolated_debt_decremented_on_liquidation`.
- **Util-on-withdraw (Blend H-03)** — `require_utilization_below_max` on borrow + withdraw + strategy; `max_utilization.rs:88`.
- **Role migration (Kamino ADV-02) + timelock bypass (Aquarius H-01) + timelock snapshot (Morpho L-06)** — full role-set migration on `accept_ownership` (`3c374e3`); both upgrade surfaces timelocked + auto-pause; per-op absolute `ready_ledger` + monotonic-non-decreasing delay.
- **Soroban hygiene** — `overflow-checks=true` + `checked_*` at money sites; no unbounded instance-storage collections (all growing registries keyed-Persistent and capped: approvals 16, pools 256, e-mode assets 64); atomic `__constructor` deploy with deployer-bound salts; no `unsafe`/`mem::forget`/blanket-`#[allow]`; soroban-sdk pinned `=26.1.0`.

## Round-2 subsystem coverage (added after completeness critic)

### `defindex-strategy` contract (production, one WASM per asset) — 0 exposed
A **non-custodial, share-free pass-through** to the controller: holds no idle balance and no internal shares; `balance()` reads `collateral_amount_for_token` live from the controller (`lib.rs:113-115`); every deposit is forwarded immediately (`lib.rs:195-201`). Consequences:
- **Share-inflation / PPS-donation / first-depositor (Spark PSM-002, Blend, Morpho)** → **n/a**. `harvest_price_per_share` derives solely from the global, oracle-free `supply_index_ray` (`lib.rs:117-125`), not from any token balance — a donation to the strategy or pool cannot move it (test `test_harvest_price_per_share_independent_of_vault_balance`). No share-mint math exists.
- **Auth-tree phishing** → **mitigated**. `authorize_supply_to_pool` builds one `InvokerContractAuthEntry` scoped to exactly `asset.transfer(strategy→pool, amount)` with `sub_invocations = Vec::new()` (`lib.rs:131-143`). `from.require_auth()` on deposit/withdraw (`:190,:235`); controller binds the strategy as account owner and enforces `require_account_owner_match`.
- **Reentrancy** → **mitigated (no guard, not exploitable)**: the contract keeps no local economic state a reentrant callee could exploit (state-of-record is the controller) and SAC tokens have no transfer hooks. **Re-evaluate if a hook-bearing token is ever listed.** (LOW residual.)
- **Fee-on-transfer** → **mitigated for SACs**: deposit returns the controller-*measured* `balance_after`, but pre-authorizes/forwards the *requested* `amount`; fine for 1:1 SACs, would mis-state for a hypothetical fee-on-transfer token. (LOW residual.)
- Note: `harvest`'s `from` is an unauthenticated event label — off-chain APY tooling must not treat it as authenticated.

### `services/keeper` (off-chain, ~3,880 LOC) — 0 exposed code findings; 1 operational residual
In scope per the corpus (Morpho-App / Radiant off-chain-signer lessons), but **off-chain** — it cannot bypass on-chain access control; blast radius = what the KEEPER-role key can do + keeper liveness + the key itself. Well-hardened:
- **Auth construction (Morpho-App)** → **mitigated**. The op (contract id + function name + args) is built locally from config (`stellar/invoke.rs:19-58`); the RPC sim fills only `soroban_data` (footprint + fees). `enforce_source_account_auth` (`stellar/tx.rs:211-246`) rejects any job whose simulated auth isn't `SourceAccount` — it never blind-attaches an RPC-returned `SorobanAuthorizationEntry`. Fails closed.
- **Signer key (Radiant blind-signing / least-privilege)** → **mitigated**. KeyVault-backed BIP-39 → SLIP-0010 hardened derivation (`signer/vault.rs`, `mnemonic.rs`); no plaintext key on disk or in committed config; key asserted strictly as **KEEPER role, never owner** (`main.rs:75-94`). The only state-mutating op is `update_indexes` (KEEPER, off by default) — **no `liquidate`/`clean_bad_debt`/`update_account_threshold`/owner-transfer/role-grant exist** (memory note `keeper_service.md` overstated the op set — correct it).
- **Idempotency (rule 2.9)** → **mitigated**: fresh sequence per submit (replay guard); TTL/restore ops naturally idempotent; retries safe.
- **Sim-vs-apply** → **mitigated**: fee headroom (`resource_fee_multiplier`, floored at `min_resource_fee`); sim error short-circuits before submit.
- **Secrets/logging (rule 2.13)** → **mitigated**: only public key + derivation path logged; no key material in logs/metrics.
- **OPERATIONAL residual (MED, deployment requirement):** a silently-dead keeper → TTL not extended → storage eviction → protocol freeze. The service exposes `tick_failed`/`entries_archived`/`tx_total` at `:9090/metrics` but does **not** self-page. **Action: the operator MUST alert on these metrics** (this is the highest-impact failure mode for the service) — alert rules now shipped in `services/keeper/ops/alerts.yml` (see #9). `mx-keyvault` supply-chain note is **resolved**: the manifest already pins the published crates.io release (`= "0.1.0"`, no git source); the stale README open-item was removed.
