# CI/CD Flow Redesign Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Eliminate duplicate CI runs on rs-lending-xlm, give Scout a Critical-only gate via a new action input, and pin the protocol to an immutable Scout tag.

**Architecture:** Two repos. (1) `scout-audit` composite action gains a `fail-on-severity` input that gates on Scout's JSON `summary.by_severity` counts; cut an immutable `v1.1.0` tag. (2) `rs-lending-xlm` drops the redundant `push` trigger from the three pure-gate workflows and configures `scout.yml` to gate on Critical, pinned to `@v1.1.0`.

**Tech Stack:** GitHub Actions (composite action + standalone workflows), bash + `jq`, Soroban/Rust toolchain (unchanged), `ruby -ryaml` for YAML validation.

- rs-lending-xlm repo: `/Users/mihaieremia/GitHub/rs-lending-xlm`, branch `main`.
- Every commit message ends with the trailer line: `Claude-Session: https://claude.ai/code/session_01Cvs8scXtrXjtNEcxfhsdtq`
- Scout severity order, high→low: `critical > medium > minor > enhancement`.
- Scout JSON gate source: `.summary.by_severity` with lowercase keys `critical|medium|minor|enhancement` (verified against a real report).
- `fail-on-severity` threshold values: `none` (default, no gate) | `critical` | `medium` | `minor` | `enhancement` | `any`. A threshold counts findings **at or above** it; `any` and `enhancement` both count all four.

- Validate every edited workflow/action YAML with: `ruby -ryaml -e "YAML.load_file('<path>'); puts 'ok'"`.

- [ ] **Step 1: Write the failing test (gate jq against fixtures)**

Create the two fixtures and the test harness that mirrors the action's gate `jq`:

```bash
SC=/private/tmp/claude-501/-Users-mihaieremia-GitHub-scout-audit/ff47f3dd-58e9-4264-bdcf-f30a584bbc2b/scratchpad
mkdir -p "$SC"
printf '%s\n' '{"summary":{"by_severity":{"critical":0,"medium":4,"minor":0,"enhancement":3}}}' > "$SC/fix_clean.json"
printf '%s\n' '{"summary":{"by_severity":{"critical":1,"medium":0,"minor":0,"enhancement":0}}}' > "$SC/fix_crit.json"
cat > "$SC/gate_test.sh" <<'SH'
#!/usr/bin/env bash
set -euo pipefail
count() { # $1 json, $2 threshold
  jq --arg t "$2" '
    (.summary.by_severity // {}) as $s |
    ($s.critical // 0)
    + (if $t=="critical" then 0 else ($s.medium // 0) end)
    + (if ($t=="critical" or $t=="medium") then 0 else ($s.minor // 0) end)
    + (if ($t=="critical" or $t=="medium" or $t=="minor") then 0 else ($s.enhancement // 0) end)
  ' "$1"
}
SC="$(dirname "$0")"
fail=0
chk() { [ "$1" = "$2" ] && echo "PASS $3" || { echo "FAIL $3 (got $1 want $2)"; fail=1; }; }
chk "$(count "$SC/fix_clean.json" critical)" 0 "clean@critical=0"
chk "$(count "$SC/fix_clean.json" medium)"   4 "clean@medium=4"
chk "$(count "$SC/fix_clean.json" any)"      7 "clean@any=7"
chk "$(count "$SC/fix_crit.json"  critical)" 1 "crit@critical=1"
exit $fail
SH
chmod +x "$SC/gate_test.sh"
```
