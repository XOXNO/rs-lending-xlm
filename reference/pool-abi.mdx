# Pool ABI

Each pool is an asset-local child contract. All mutating pool functions are admin-gated by owner auth, and the owner is the controller.

The stable interface is declared in `pool-interface/src/lib.rs`.

## Constructor

```rust
__constructor(admin: Address, params: MarketParams)
```

Initializes pool owner, stores `MarketParams`, and creates `PoolState` with:

```text
supplied_ray = 0
borrowed_ray = 0
revenue_ray = 0
borrow_index_ray = RAY
supply_index_ray = RAY
last_timestamp = ledger.timestamp() * 1000
```

## Mutating accounting calls

| Function | Returns | Purpose |
| --- | --- | --- |
| `supply(position, price_wad, amount)` | `PoolPositionMutation` | Syncs indexes, adds scaled supply, returns updated position and indexes. |
| `borrow(caller, amount, position, price_wad)` | `PoolPositionMutation` | Syncs indexes, checks reserves, adds scaled debt, transfers asset to caller. |
| `withdraw(caller, amount, position, is_liquidation, protocol_fee, price_wad)` | `PoolPositionMutation` | Syncs indexes, burns scaled supply, applies liquidation fee when requested, checks reserves, transfers net asset. |
| `repay(caller, amount, position, price_wad)` | `PoolPositionMutation` | Syncs indexes, burns scaled debt, refunds overpayment to caller. |
| `update_indexes(price_wad)` | `MarketIndex` | Syncs interest and returns latest indexes. |
| `add_rewards(price_wad, amount)` | `()` | Syncs indexes and increases supply index with reward amount. |
| `create_strategy(caller, position, amount, fee, price_wad)` | `PoolStrategyMutation` | Opens strategy debt, records fee as revenue, transfers net amount to caller. |
| `seize_position(position, price_wad)` | `AccountPosition` | Clears deposit dust into revenue or bad debt into supply-index socialization depending on position type. |
| `claim_revenue(price_wad)` | `i128` | Burns claimable scaled revenue and transfers asset to pool owner. |

## Flash-loan hooks

| Function | Purpose |
| --- | --- |
| `flash_loan_begin(amount, receiver)` | Syncs indexes, checks reserves, snapshots pool balance, transfers amount to receiver. |
| `flash_loan_end(amount, fee, receiver)` | Pulls amount plus fee from receiver, verifies post-balance, records fee as revenue. |

The pool ignores caller-supplied asset IDs for flash loans. It always uses `params.asset_id`.

## Admin update calls

| Function | Purpose |
| --- | --- |
| `update_params(max_borrow_rate, base_borrow_rate, slope1, slope2, slope3, mid_utilization, optimal_utilization, reserve_factor)` | Syncs indexes, validates monotone rate curve, updates mutable rate params. |
| `upgrade(new_wasm_hash)` | Upgrades pool WASM. |
| `keepalive()` | Extends pool instance TTL. |

## Views

| Function | Returns | Meaning |
| --- | --- | --- |
| `capital_utilisation()` | `i128` | Current utilization. |
| `reserves()` | `i128` | Live token balance held by pool. |
| `deposit_rate()` | `i128` | Current supply rate. |
| `borrow_rate()` | `i128` | Current borrow rate. |
| `protocol_revenue()` | `i128` | Reconstructed claimable revenue. |
| `supplied_amount()` | `i128` | Reconstructed total supplied amount. |
| `borrowed_amount()` | `i128` | Reconstructed total borrowed amount. |
| `delta_time()` | `u64` | Elapsed milliseconds since last sync. |
| `get_sync_data()` | `PoolSyncData` | Full `MarketParams` plus `PoolState`. |

## Defense-in-depth checks

Every mutating pool path includes local validation so a future controller bug cannot silently corrupt pool state:

- Rejects negative amounts.
- Verifies position asset matches `params.asset_id`.
- Checks reserves before outgoing transfers.
- Guards scaled subtraction underflows.
- Validates rate model on pool-side updates.
- Uses pre/post balance check for flash-loan repayment.

## Controller dependency

The controller depends on `pool-interface`, not the `pool` crate. That keeps runtime coupling on ABI only and makes the controller-pool trust boundary explicit.
